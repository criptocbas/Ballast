01The composition

Jupiter built Prediction as a speculation product and Lend as a yield product. Ballast composes them as the underwriting layer and the premium engine of a yield-bearing insurance vault — a flip neither product was designed for, but which the public APIs make possible. Depositors never directly pay a premium; the vault self-insures with native Jupiter primitives.

02System

Three packages, one daily cron, five Jupiter APIs. Custodial v1 — single keypair, server-side only.

Custody: v1 is explicitly custodial. A single Solana keypair (the "vault keypair") holds all deposits and Jupiter positions. Private key in env var, server-side only — never logged, never in any NEXT_PUBLIC_*, never in any apps/web file. Documented at length in ARCHITECTURE.md; v2 path is an on-chain vault program issuing share tokens.

03Rebalance loop

The heart of the system. One cron tick a day at 00:00 UTC, plus an admin-triggered manual run for demos. Cooldown is persisted to SQLite (vault_state table) so a crash inside the cooldown window cannot bypass the gate.

1

Acquire cooldown

Read vault_state['rebalance.last_tick_started_at_ms'] from SQLite. If within the 1-hour cooldown window, abort. Otherwise mark the tick start and proceed.

2

Refresh state

Read Lend Earn position, current hedge marks, depositor table, vault USDC + SOL balance. Read paths use withRpcFallback(); primary RPC fails over to SOLANA_RPC_URL_FALLBACK on rate-limit.

3

Withdraw accrued yield

Compute share growth on Lend Earn since last tick. Withdraw the delta via POST /lend/v1/earn/withdraw. Tx is built/signed/sent through buildSimSignSendConfirmV0 — up to 2 attempts on BlockhashExpired with a fresh blockhash. Fired in production on 2026-05-07 on a $12.50 deposit.

4

Settle resolved positions

claimer.ts walks open hedges, calls POST /prediction/v1/positions/{id}/claim on any resolved-winner contract. Jupiter-supplied transactions are sent as-is — wrapping them in our retry helper would re-sign under a fresh blockhash and break the keeper contract.

5

Convert payouts to USDC

Any non-USDC settlement (e.g. SOL claim residue) goes through Swap V2 (/swap/v2/order → /swap/v2/execute) to land everything in USDC for accounting.

6

Allocate

Split freed USDC by HEDGE_BUDGET_FRACTION (default 50%): half compounds back into Lend Earn, half feeds the hedge bucket.

7

Place hedges

Walk basket.config.json. Allocate per-market hedgeBudget × weight. Skip if < $5 (Jupiter Prediction's per-order minimum — DX-GAP-#18, #24). Skip markets the vault already holds. Place BUY NO via POST /prediction/v1/orders. Persist the new hedge row.

live demo: 1-market basket (POLY-1345531) v1.5: 5-market diversified (preserved in JSON)
8

Persist

Append to event log. Write the new tick timestamp to vault_state. Bust the in-memory /vault/info cache so public reads reflect new state immediately. Release the cooldown.

Decoupled withdrawal worker

Independent cron (WITHDRAWAL_PROCESS_CRON, default every 10 minutes) processes queued depositor withdrawals. Race-safe: trySettleWithdrawal uses an atomic UPDATE … WHERE status='pending' with a changes === 0 check so two workers cannot double-settle. Decoupled so daily-cron crashes don't starve withdrawals.

05Robustness layer

Six concrete improvements added 2026-05-06 before the demo. All tested. One — the blockhash-expired retry — fired in production the next day on a real $12.50 deposit and recovered cleanly.

1. Persistent rebalance cooldown

vault_state KV row (rebalance.last_tick_started_at_ms). Survives orchestrator restart by design — a crash inside the cooldown window cannot bypass the gate.

2. Decoupled withdrawal worker

Independent 10-minute cron. Atomic UPDATE … WHERE status='pending' soft-lock; changes === 0 on contention.

3. Cache bust on admin mutations

bustVaultInfoCache() after every /admin/* write so public /vault/info reads reflect new state instead of serving 15s-stale data.

4. Multi-RPC fallback

SOLANA_RPC_URL_FALLBACK via withRpcFallback() for read paths. Write paths intentionally don't fall back — blockhash freshness contract.

5. Per-route rate limit

/api/me/:wallet capped at 60 req/min/IP. Wrapped in a sub-plugin to satisfy Fastify's onRoute hook ordering — a 20-minute lesson.

6. Blockhash-expired tx retry

buildSimSignSendConfirmV0 in tx.ts replaces ~80 lines of inline build/sign/send/confirm. Up to 2 attempts with fresh blockhash. Real-world validated.

06Custody & security boundaries

v1 ships custodial. Mitigations in place; v2 path documented.

Boundary Mitigation
Vault keypair compromise Stored in env var only; never logged; never exposed to frontend; rotation procedure documented. Catastrophic if lost — entire trust surface.
Frontend → orchestrator API Sign-message auth: server-issued nonce → client signs canonical message ballast:<purpose>\nnonce=<n>\n<sorted-bindings> → server verifies via tweetnacl, one-shot consume. Replay-resistant.
Admin endpoints Authorization: Bearer $ORCHESTRATOR_ADMIN_TOKEN with constant-time comparison. 503 if env unset; 401 on mismatch. Fastify error handler passes 4xx through with structured statusCode (rate-limit 429 reaches the client correctly).
Withdrawal flow Signed intent → orchestrator validates against withdrawableNow (DX-GAP-#28 fix), not notional → settles inline if redeemable, else queued. Capped UI at min(notional, shareFraction × redeemable).
PII surface /vault/aggregate is the public depositor surface (count + total only). /admin/depositors is the full list. /api/me/:wallet rate-limited at 60 req/min/IP.
Bad-actor markets Curated basket via config file; not user-selectable in v1. Each market entry has a written thesis; closed markets must be re-curated before restore.
